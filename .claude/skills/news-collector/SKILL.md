# AI News Collector

## ワークフロー

1. `config/whitelist.json` を読み込み、ソース URL 一覧を取得
2. 各 URL に対して WebFetch を実行し、ページコンテンツを取得
3. 記事メタデータを抽出（タイトル、URL、公開日時、ソース名）
4. カテゴリ分類を実行
5. 重要度判定を実行
6. 日本語で概要（100 文字程度）を生成
7. 一意の ID を付与（形式: YYYYMMDD-NNN）
8. `data/current.json` に保存し、`lastUpdated` を更新

## カテゴリ分類ルール

| カテゴリ | 判定条件 |
|----------|----------|
| Model | LLM、基盤モデル、学習手法、モデルアーキテクチャ、ベンチマーク結果に関する記事 |
| Service | API、SDK、製品、サービス、料金、機能追加に関する記事 |
| Other | 上記以外（人事、パートナーシップ、イベント、ポリシー等） |

## 重要度判定ルール

| 重要度 | 判定条件 |
|--------|----------|
| high | 新モデル発表、メジャーバージョンリリース、大型買収（10億ドル超）、業界初の技術発表 |
| normal | 上記以外すべて |

## 出力データ形式

```json
{
  "lastUpdated": "2025-12-25T06:00:00Z",
  "news": [
    {
      "id": "20251225-001",
      "title": "記事タイトル",
      "url": "https://example.com/article",
      "sourceName": "OpenAI Blog",
      "category": "Model",
      "publishedAt": "2025-12-25T00:00:00Z",
      "summary": "記事の概要（日本語、100文字程度）",
      "importance": "high"
    }
  ]
}
```

## エラーハンドリング

- URL アクセス失敗: ログ記録してスキップ、次の URL を処理
- メタデータ抽出失敗: 可能な範囲で抽出、不明項目は空文字
- 重複記事検出: ログ記録してスキップ
